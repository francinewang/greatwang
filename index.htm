About This Document

The GVT (Globalization Verification Test) is an essential phase in the software development cycle, meant to ensure the basic global functionalities including support for languages, character sets, and various kinds of culturally specific formatting rules.


The GVT Guide provides information on the test areas that may need be covered in a GVT, and essential testing techniques with step-by-step instructions that the testers should follow. It also provides the test data that is needed. With the techniques explained in this document and the test data, the testers don't necessarily have to be native speakers of the target languages, and they can still do as well as the native speakers working as globalization specialists.

The primary entry points to the GVT Guide are determined by the GVT Wizard. The GVT Wizard identifies what may need testing and provides deep links into this GVT Guide which explains how each test area can be tested. This GVT Guide contains numerous tests that may not be at all relevant to your software and so you should not assume that all the tests presented here should be performed on your software.

1. Introduction

The objective of a GVT is to verify the globalization function enablement so that we deliver quality software to the worldwide market. The GVT must be performed prior to the TVT (Translation Verification Test), and is recommended to be done as a part of the FVT (Functional Verification Test). These process requirements are based on many years of experience with worldwide General Availability (GA) releases for many products. All IBM software development teams are required to perform a GVT, and the completion of the GVT is an entry criterion for the TVT.

In the proper development process, the global functions of the software should be implemented as prescribed in the "Globalization White Paper", which defines the basic language support required for each software family and software type.

2. GVT Resources

The following GVT resources are available along with the GVT Guide.

  • GVT process web site
  • GVT Wizard

The GVT process page describes the IBM GVT process from end to end and includes a GVT plan template. The GVT Wizard, a web-based program, provides a Question and Answer (Q&A) methodology which enables the user, such as a product test manager, tester or test leader, to identify the minimal set of globalization test areas that are applicable to software.

3.Test Areas

Each of the test areas described here has three kinds of related information, a title of the test area matching the test area titles used in the GVT Wizard, a description of the test operation, which guides you through the testing steps, and some test data, which you can copy and paste, or use as an example to complete the tests.

Repetition of tests in multiple locales
The GVT Guide documents when specific tests need to be repeated in different locales and also states exactly which carefully selected locales are required for such test areas. Unless the GVT Guide states that a test must be done in a specific locale, any locale can be used. If no locale is specified by the GVT Guide, and if your software interacts with multiple machines, it is a good practice to have the machines all running in different locales and not using the locale predominately used by the developers.

Minimum coverage
The test data and scenarios have been chosen in order to provide the minimum adequate coverage for the specific test area. In order to perform adequate coverage of a test area you should do all the tests that are described unless the test area clearly provides alternate test data to handle situations where only one set of data cannot be used for all programs.

Test cases
The GVT Guide does not provide test cases, it provides the test data and generic steps needed in order to perform a test. The GVT Guide provides the generic test approach and the test data. Actual test cases that are specific to your software need to be produced based on the GVT Guide. It is the responsibility of the test team to produce specific test cases for your software based on the steps and test data provided here.

Feedback
We are interested in your feedback, especially if you think that we have an error or have missed something out. Creating generic test steps for all software is a very challenging task and the GVT Guide is still relatively new and pages are still being added. Please send questions and comments to Dale Schultz/Westford/IBM.

Structure
There are six major test area categories based on the functional aspects of software. Each test area has one or more links to sub-documents in which each test area is described in detail. In order to identify the applicable test areas of the software to be tested, the GVT planners or testers should use the GVT Wizard first. The GVT Wizard will provide a report containing the identified test areas and will include links to each test area in this guide.

Note: The GVT Guide does not define a test case, it provides generic guidance for the creation of test cases that are specific to the software under test. There may be multiple places in the software being tested that need similar test cases. Take care when copying test data from the GVT Guide to your test case repository so that it does not get corrupted by your test case storage systems. It is advisable to store a date and link to this page in your test cases so that you can check for updates to the GVT Guide in future test cycles of your software.
